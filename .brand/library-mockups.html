详情页 V2 学术风设计要点
· 顶部品牌:大字 serif "Deeply" + 副标题(每本书一句话定位),代替普通 nav-bar 标题。沉浸感强。
· 排版:Noto Serif SC 中文衬线 + Playfair Display 英文衬线,致敬学术 / 出版调性。
· 「TODAY'S ECHO」:浅黄边框小卡,一句"为什么这本书今天还重要"。这个是静态文案,不是每日生成,可手工写或 LLM 预生成,不是"今日精选"。
· 知识谱系:放大成独立区块(米黄背景区分),分「这本书继承自」/「这本书启发了」两段,横滚卡片。每张卡:关系 chip(继承/启发/回应/批评/改造)+ 小封面 + 书名+作者 + 「为什么相关」一句话。可点击 → 跳到该书详情,形成"读完一本 → 想了解它的祖先 / 后裔 → 进入下一本"的连续 explore。
· 「探索完整图谱」:底部入口,跳进可视化网络图(34819 本书的全景),给愿意深挖的用户开窗。

为什么不直接 port 桌面力导向图
桌面 ECharts 力导向图(deeply.html 那个)在小屏失灵:
  • 节点重叠看不清书名
  • 手指拖动 / 缩放 / 点中节点都很难
  • 14.7w 边一次性渲染卡死
手机端要"碎片化进图谱",每次只看局部,跟"看长图"完全不同。
三段式设计
① Ego-network:以当前书为中心,8-12 个一跳邻居围绕排列。节点大小 = PageRank,边颜色 = 关系类型。点节点跳到该书详情,中心切换重新布局。
→ 用户感受:"史记跟它的邻居们的样子"

② 推荐阅读路径:从图算法预生成的 3-5 条 N 跳路径,每条卡片显示节点链 + 一句话推荐 + "总 N 节 · 约 N 小时"。
→ 用户感受:"沿这条路读下去会看到什么"
→ 这是把图谱"线性化",让"看图"变成"按一条路走"。这种方式比"在图上瞎逛"心智负担小得多。

③ 全图入口:深色卡片,带按维度切的 chip(时代/学科/影响力/国家)。点进去是真的全图浏览页(那是另一个 surface,有需要再设计)。
数据 / 工程
· Ego-network:filter `edges_merged.json` 里 from=当前书 或 to=当前书 的边,top by 邻居 PageRank,取 8-12 个。SVG 静态渲染就够;真要交互就用 D3 / react-native-svg。
· 阅读路径:这是核心算法。可以是:
  a) 预生成 — 对 PageRank top 500 本书离线跑 BFS / 主题路径算法,每本配 3-5 条;
  b) 实时 — 用户点进图谱页时,从 edges 即时算路径,LLM 写一句"为什么这条";
  c) AI 端到端 — LLM 看当前书 + 全图 metadata,直接出"3 条推荐路径"。
建议 MVP 走 (a),热门书离线预生成。
· 全图:暂时打到一个 web view 跳转 deeply.html 类的桌面版,后期再做移动原生大图。
心智模型 ③ 详情页里的"知识谱系"小区 = 这本书直接的 1 跳邻居。
④ 图谱页 = 同样的邻居 + 沿这本书的多跳路径 + 全图入口。
两者关系:③ 是"用一句话讲清单本书在图谱里的位置";④ 是"把图谱本身打开"。
