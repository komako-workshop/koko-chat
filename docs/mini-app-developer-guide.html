KokoChat Mini-App Developer Guide

面向第一批内置小程序开发者：UI 在 KokoChat，能力在 OpenClaw。小程序只声明自己需要什么 agent、skills、core tools，然后用 KokoChat runtime 调用。

心智模型

KokoChat 负责

  • 移动端 UI 与导航
  • Conversation 列表和聊天壳
  • mini-app registry
  • OpenClaw Gateway 连接
  • runtime API: inferOnce / agent session

OpenClaw 负责

  • agent 运行
  • skill / tool 能力
  • Mac 端网络与本地文件
  • shell / browser / web fetch 等能力
  • 长期 transcript 与工具副作用
第一版小程序仍然是内置小程序，随 KokoChat 打包。未来可以走 WebView mini-app runtime，但现在不要动态加载 RN 代码。

推荐开发流程

1. 复制 example 模板
apps/koko-chat/sources/miniapps/example/
2. 注册 mini-app descriptor
声明 id、displayName、openclaw.defaultAgentId、requiredSkills、requiredCoreTools。
3. 选 OpenClaw 能力集
先从 core tools 中选最小集合，再加本地 skill 文件夹。
4. 写 UI / outbound / blocks
KokoChat 端负责用户体验和结果渲染。
5. 用 Self-Test 和真实 mini-app 流程回归
先确认 Gateway、inferOnce、agent session 正常。

Mini-App Descriptor

Descriptor 是小程序的声明入口。它让 host 知道如何显示、创建和路由这个 mini-app。

registerMiniApp({
  id: "tavern",
  displayName: "Tavern",
  showInLauncher: true,
  listGlyph: "🍺",
  defaultTitle: (createdAt) => `Tavern ${formatTime(createdAt)}`,
  openclaw: {
    defaultAgentId: "tavern",
    requiredSkills: ["kokochat-tavern-search"],
    requiredCoreTools: ["web_search", "web_fetch"],
    localSkillDirs: ["openclaw/skills/kokochat-tavern-search"]
  }
});
字段含义
idmini-app id，也是 conversation mode。
displayName显示在 + 菜单和开发工具中的名称。
showInLauncher是否出现在会话列表 + 菜单。
listGlyph会话列表无头像时的 glyph。
defaultTitle未传 title 时的默认会话标题。
openclaw.defaultAgentId该 mini-app 默认调用的 OpenClaw agent。
openclaw.requiredSkills该 agent 需要可见的 optional skills。
openclaw.requiredCoreTools该 agent 需要的 OpenClaw core tools。
openclaw.localSkillDirs随 repo 提供的本地 skill 文件夹。

Agent 默认规则

  • claw 默认使用 main agent。
  • 其他 mini-app 默认使用同名 agent，例如 tavern -> tavern。
  • descriptor.openclaw.defaultAgentId 可以覆盖。
  • inferOnce / createAgentSession 显式传 agentId 优先级最高。
agent:main:kokochat:claw:<scope>
agent:tavern:kokochat:tavern:<scope>
不要让产品 mini-app 默认污染 main。main 是用户的“家”，mini-app agent 是具体产品工作间。

Skills 与 Core Tools

Core tools 是 OpenClaw 内建工具，如 web_fetch、read、exec。Skills 是可选能力包，通常由 SKILL.md 和可执行脚本组成。

类型稳定性适合做什么
Core tools更像基础设施，但具体 profile/policy 仍需确认Web、文件、shell、session、memory、media 等通用能力
Optional skills用户安装情况不同产品特定能力，如 kokochat-tavern-search

原则：mini-app agent 使用明确的能力子集。不要默认继承 main agent 的全部 skills。

推荐能力集

Mini-App 类型建议 Core Tools建议 Skills
纯聊天session_status 或 none产品 prompt/skill
搜索/发现web_search, web_fetch一个业务 skill，如 kokochat-tavern-search
本地导入/解析read, maybe imageparser/import skill
Codex/Manus 类read, write, edit, apply_patch, exec, process, sessions_spawn, update_plancoding/product skills
定时/后台cron, session_statustask skill

Runtime API

一次性推理

const result = await inferOnce({
  miniAppId: "tavern",
  prompt
});

这会走默认 agent 规则，生成临时 session，等待完成，读取结果，并删除临时 session。它不是纯 LLM API，而是一次临时 OpenClaw agent 调用。

长期 Agent Session

const session = await createAgentSession({ miniAppId: "tavern", scope: conversationId });
const send = await sendAgentMessage({ sessionKey: session.key, message });
await waitForAgentRun({ runId: send.runId });
const history = await readAgentHistory({ sessionKey: session.key });

Storage

const storage = getMiniAppStorage("tavern");
storage.setJson("settings", { safety: "sfw" });
const settings = storage.getJson("settings");

所有 key 都在 mini-app namespace 下。不要直接写裸 MMKV key。

Conversation

const meta = useConversationStore.getState().create({
  mode: "tavern",
  title: "Tavern",
  listSnapshot: { title: "Tavern", subtitle: "Find characters" }
});

ConversationMeta 只放 mode、sessionKey、artifactRef、listSnapshot 等轻量元信息。完整业务数据放 mini-app storage 或 OpenClaw 侧数据目录。

Outbound Hook

当前 outbound builder 同时承担“改写发送内容”和“拦截本次发送”的角色。若返回 localOnly: true，host 不发 Gateway，mini-app 自己负责写 agent 回复。

registerOutboundMessageBuilder("tavern", async ({ visibleText, conversation }) => {
  runMyFlow(conversation.id, visibleText);
  return { visibleText, gatewayText: visibleText, localOnly: true };
});
这个 API 后续可能会拆成 interceptUserSend 与 buildOutboundMessage。

Message Blocks

用于在聊天流中渲染结构化卡片。模型输出必须先经过 mini-app parser/validator，再变成 block data。注册 renderer 时推荐提供 guard，host 会先校验 block.data，失败则显示 fallback。

registerSharedBlockRenderer(
  "koko.tavern.character_card",
  isCharacterCardData,
  CharacterCardRenderer
);

从 agent 文本里提取 fenced block 时使用共享工具：extractFencedBlock(text, "koko.tavern.recommendations")。它只负责提取 body，不负责 JSON parse 或 schema 校验。

给开发窗口的话

Tavern 小程序直接走 agent + skill 架构，不做 prompt-only demo。注册 descriptor 时声明 id: "tavern"、openclaw.defaultAgentId: "tavern"、requiredSkills: ["kokochat-tavern-search"]、requiredCoreTools: ["web_search", "web_fetch"]、localSkillDirs: ["openclaw/skills/kokochat-tavern-search"]。调用时使用 inferOnce({ miniAppId: "tavern", prompt })，不要显式传 agentId: "main"。KokoChat 端只收集用户需求、调用 tavern agent、解析/校验结构化结果并渲染角色卡；搜索 character-tavern.com、抓取、解析、归一化都属于 kokochat-tavern-search skill/tool。
